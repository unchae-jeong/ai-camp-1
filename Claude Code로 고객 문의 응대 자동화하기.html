기존 루틴 vs 자동화 루틴

단계Before (수동)After (자동화)
1. 문의 확인 Gmail 열어서 직접 확인 inbox-alert.sh가 15분마다 감지 → Slack 알림
2. 분류 머릿속으로 카테고리 판단 /categorize로 3단계 자동 분류
3. 가이드 검색 help.airbridge.io 수동 탐색 /guide-digest로 키워드 자동 매칭
4. 답변 작성 이전 메일 찾아 재작성 또는 처음부터 카테고리+가이드 기반 초안 자동 생성 → 검토
5. 아침 정리 출근 후 Gmail 훑으며 수동 정리 daily-inbox-triage.sh로 Obsidian에 자동 정리
6. 패턴 파악 감으로 판단, 정량 근거 없음 카테고리 통계 자동 집계 → 업무 우선순위 조정

/guide-digest — 가이드 검색 & 요약

분류 키워드로 help.airbridge.io 공식 가이드를 자동 검색하고 한국어로 요약

동작 방식

  1. Sitemap 기반 인덱스 자동 구축
    7일 주기 자동 갱신
  2. 한영 키워드 매핑 (28개)
    메타→meta, 딥링크→deeplink
  3. 인덱스 매칭 → 본문 수집
  4. 한국어 요약 + 참고 링크 첨부

실행 예시

/guide-digest 메타 연동 매칭: meta, channel-integration 가이드: 채널 연동하기 - 메타 애즈 요약: Meta 전환 API 연동 시... 링크: https://help.airbridge.io/...

답변 초안 생성

카테고리 + 가이드 = 답변 초안 — CSM이 검토 후 발송

Airbridge 대시보드 > 채널 연동 > Meta 설정에서 ... 2. Meta 이벤트 매니저에서 서버 이벤트 수신 여부 확인 ... [참고 가이드]: https://help.airbridge.io/hc/ko/articles/... [참고 소스]: Help Guides

정보 추출 누가, 어떤 솔루션의, 어떤 카테고리를 기준으로 문의했는지 정리 문의 주체(고객사/대행사), 회사명, 3단계 분류, 요약, 긴급도, 키워드까지 자동 추출
